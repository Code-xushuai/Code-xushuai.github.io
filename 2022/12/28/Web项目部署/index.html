本质三件事

  1. 租服务器 + 配置环境
    (1) 使用Centos

     - 部署 linux系统

    (2) 公网IP
    (3) 登录服务器 (一般远程连接服务器)

     - SSH工具
     - win（xshell）
         ssh root@公网IP   
     - mac (item2)

    (4) 关于端口

     IP地址 ---> 电脑
 端口   ---> 各种各样的程序和服务
 22   : SSH
 3306 : MySQL
 6379 : Redis
 80   : (Http)编写网站
 443  ：HTTPS

    (5) 安全组 （配置端口是否允许）

     租服务器的网站上配置

  2. 代码放在服务器中
    (1) git
    (2) git代码仓库：github、gitee、gitlab

     - 一般创建代码仓库名与本地项目文件名一致

 - 不需要上传的文件需要在项目文件的下配置.gitignore


     - 仓库地址：https://gitee.com/code-xs/django_file

    (5) git命令

     - 一次性命令（本地与仓库连接：配置）
     >>> git config --global user.name "上传人的名字"
     >>> git config --global user.email "上传人的邮箱""

     >>> cd day28 ：进入项目文件下打开git
     >>> git init : 初始化
     >>> git remote add origin 仓库地址
 - 上传到仓库
     >>> git add .
     >>> git commit -m "init" : " "为备注信息
     >>> git push origin master

    （6）下载代码（下载到服务器）

     使用工具远程连接服务器xshell

 - 安装git
     >>> yum install git -y
 - 第一次下载代码
     >>> cd /data/
     >>> mkdirs www : 在data文件下创建www文件
     >>> cd /data/www/ 
     >>> git clone 仓库地址 
     >>> 码云 账户+密码
 - 之后需要更新代码
     >>> cd /data/www/day28/ : 需要进入代码文件下
     >>> git pull origin master

  3. 程序运行起来
    使用工具连接服务器（服务器上进行安装）
    (1) 安装Python对应的版本

     - 安装gcc，用于后续安装Python时编译源码。
         >>> yum install gce -y

    (2) 安装Python3相关依赖 

         >>> yum install zlib zlib-devel -y
     >>> yum instal1 bzip2 bzip2-devel -y
     >>> yum install ncurses ncurses-devel -y
     >>> yum install readline readline-devel -y
     >>> yum instal1 openssl openssl-devel -y
     >>> yum instal1 xz lzma xz-devel -y
     >>> yum install sqlite sqlite-devel -y
     >>> yum instal1 gdbm gdbm-devel -y
     >>> yum install tk tk-devel -y
     >>> yum install mysql-devel -y
     >>> yum instal1 python-devel -y
     >>> yum insta1l libffi-devel -y

    (3) 下载源码 Python

         >>> yum install wget -y

     >>> cd /data/  : 将Python放在data目录下
     >>> wget https://www.python.org/ftp/python/3.9.5/Python-3.9.5.tgz

    (4) 解压 & 编译 & 安装

         >>> tar -xvf Python-3.9.5.tgz  ： 解压

     要进入目录在编译
     >>> cd Python-3.9.5 
     >>> ./configure
     >>> make all
     >>> make install

    (5) Python解释器配置豆瓣源

         >>> pip3.9 config set global.index-url https://pypi.douban.com/simple/

  4. 安装虚拟环境
    (1) 安装virtulenv

     >>> pip3.9 install virtualenv

    (2) 创建虚拟环境

     代码位置：
         /data/www/day28
 环境位置：
         /envs/nb

 创建存储环境的文件夹：
         >>> mkdir /envs  : 在根目录下创建envs文件夹
         >>> virtualenv /envs/nb --python=python3.9

    (3) 激活虚拟环境 

     >>> source /envs/nb/bin/activate

  5. uwsgi（更新代码需要重新启动uwsgi）
    (1) 安装uwsgi

     1.前提需要先激活虚拟环境 
 source /envs/nb/bin/activate
 2.在安装wusgi
 pip install uwsgi

    (2) 基于uwsgi运行flask项目

     配置文件（推荐）
 1. 创建nb_uwsgi.ini
     - 现在本地的项目跟目录下创建nb_uwsgi.ini文件
     - 在上传到仓库
     - 浏览器拉下来
 2. 配置文件
 nb_uwsgi.ini
     [uwsgi]
         socket = 127.0.0.1:8001
         chdir =/data/www/day28  ：项目文件
         wsgi-file = app.py
         cal1able = app
         processes = 1     : 进程（几核几个进程）
         virtualenv = /envs/nb/   : 虚拟环境位置
 3. 执行命令 
     >>> source /envs/nb/bin/ activate
     >>> uwsgi --ini nb_uwsgi.ini &   ： 启动uwsgi（加&为后台一直启动）
     ![需要找到进程号](image/94.png)

  6. Nginx
    (1) 安装Nginx

     >>> yum install nginx -y

    (2) 配置

     - 普通请求     ->  8001端口（找到uwsgi监听的端口）
 - /static/     ->  /data/www/day28/static (找到static静态文件)

 nginx的默认配置文件路径  /etc/nginx/nginx.conf 
 - 删除默认的nginx.conf   (需要在虚拟环境激活前提下)
     >>> rm nginx.conf

 - 新建nginx.conf  + 内容拷贝  + 保持
    >>> cd /etc/nginx/
    >>> vim nginx.conf   : 创建并打开文件（前提需要在/etc/nginx/目录下，vim相当于一个记事本）
    >>> i             ：变换成编辑模式 
    >>> ESC(按下)     ：退出编辑模式
    >>> :wq           : 保持并退出
    >>> cat nginx.conf     : 查看nginx.conf文件

    (3) 启动 (接下来就需要启动uwsgi和nginx：)

     - 启动
     systemctl start nginx
     systemctl stop nginx
     systemctl restart nginx
 - 开机启动
     systemctl enable nginx

    (4) nginx,.conf配置文件

  7. Shell脚本 （在项目根目录下创建这两个脚本）
    (1) 重启 reboot.sh

    (2) 停止 stop.sh

    (3) 上传到仓库中，更新到浏览器中
    (4) 两个脚本赋予可执行权限

     >>> chmod 755 reboot.sh
 >>> chmod 755 stop.sh

    (5) 执行

     启动
 >>> ./reboot.sh
 停止
 >>> ./stop.sh

  8. 项目依赖Mysql服务
    (1) 安装MySQL服务

     >>> yum install mariadb-server -y    服务端
 >>> yum install mariadb -y           客户端

    (2) 服务配置

     启动
     >>> systemctl start maridb
     >>> systemctl stop maridb
 开机启动
     >>> systemctl enable maridb

    (3) 授权

     >>> mysql -uroot -p
 2、创建数据库  day28db
     >>> CREATE DATABASE day28db DEFAULT CHARSET utf8 COLLATE utf8_general_ci;
 3、创建账户   s5
     >>> insert into mysql.user(user,host,password) values( 's5' , '%' , password(' qwe123..'));
     >>> flush privileges;
 4、授权
     >>> grant all privileges on day28db.* to s5@'%';
     >>>  flush privileges;
 5、测试
     - 远程测试
         使用navicat远程连接服务器ip
     - 本地测试
         无密码登录

Https部署


uWSGl

uWSGI是一个Web服务器，它实现了WSGl协议、uwsgi、http等协议。Nginx中HttpUwsgiModule的作用是与uWSGI服务器进行交换。

WSGI

Web服务器网关接口(Python Web Server Gateway Interface，缩写为WSGI)
WSGl是一种通信协议,Flask,webpy，Django、CherryPy等等都自带WSGl，不过性能都不好。

uWSGI

uWSGI同wSGI一样是一种通信协议。
而uWSGl是实现了uwsgi和WSGI两种协议的Web服务器。

Nginx、WSGI、uWSGI之间的关系

uWSGl在web服务器与web框架之间作用: uWSGl就像一条纽带，将web服务器与web框架连接起来。Nginx属于一种高并发能力web服务器，Flask属于一种web框架，因此，uWSGl与Nginx、Flask 的作用就不明而喻
