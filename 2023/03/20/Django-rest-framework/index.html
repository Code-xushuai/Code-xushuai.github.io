# Restful api

/book/ GET 查看所有资源，返回所有资源
/book/ POST 添加资源，返回添加的资源
/book/1 GET 编辑某个资源，返回这个资源
/book/1 PUT 编辑某个资源，返回编辑之后的这个资源
/book/1 DELETE 删除某个资源，返回空

  • 视图类

  • dispatch 函数首先执行

# APIView

  • 导包
    from rest_framework.views import APIView

  • 与 view 区别
    重新声明了一个新的 as_views 方法并在 dispatch 进行路由分发前，会对请求的客户端进行身份认证、权限检查、流量控制

  • 常用属性

    • .data
      request.data: 返回解析之后的请求体数据。类似 django 中标准的 request.POST 和 request.FILES 属性，但提供一下特征：
      1）包含解析之后的文件和非文件数据
      2）包含 POST、PATCH、PUT 请求方式解析之后的数据
      3）利用 parsers 解析器，不仅支持表单数据，也支持 json 数据
    • .query_params
      request.query_params 与 django 中的 request.GET 相同

# serializers 序列化器

  • 前端接收后端：序列化 (后端压成 json 发给前端) （GET）
    后端接收前端：反序列化（前端传 json 数据） (POST)

  • 导包
    from rest_framework import serializers

  • 注意

    • 针对 model 模型设计序列化器
      【views】
    • 在序列化器类中，进行字段的序列化
    • 构建序列化对象，传入需要序列化对象（数据库查询的对象）
    • 返回值：使用 rest_framework 中的 Response

  • 关联序列化
